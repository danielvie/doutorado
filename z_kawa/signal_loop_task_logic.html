signal_loop_task Logic

This page documents the current signal generation logic in main/src/signal_controller.cpp. It is written as both reference documentation and a tutorial for how the component turns dataset segments into three synchronized digital outputs with dead-time protection.

Execution coreContinuous playback runs in signal_loop_task, pinned by signal_start_continuous() to Core 1.
Timing unitThe loop stores durations as 0.1 us ticks and converts them to CPU cycles with SIGNAL_CYCLES_PER_TIME_TICK = 24.
Output modelEach logical channel controls a LOW pin and a HIGH pin. Only one side is enabled after dead time.

Output Mapping

The dataset stores three mode bits per segment. During precompute, those bits are translated into GPIO masks so the hot loop can write raw GPIO registers without branching over pin names.

How to read the chart

  • Each segment chooses one state for U1, U2, and U3 from the mode bits.
  • A high line means the HIGH-side pin is active. The matching LOW-side pin is active when the line is low.
  • The red bands are transition windows where changed channels are cleared before the next pin state is asserted.
  • Unchanged channels are included in the final set/clear masks, but they do not require the initial dead-time clear.

Dead-Time Detail

Dead time is inserted only when a channel changes state. During signal_precompute_steps(), the new mode is compared against the previous segment. The previous segment for index 0 is the final segment, so the repeated pattern has the same protection at the wraparound boundary.

The execution order inside execute_signal_pattern() is:

  1. Add the segment duration to next_edge. This makes the schedule absolute instead of delay-chained.
  2. If the segment is shorter than dead_time + MIN_ACTIVE_HOLD_CYCLES, clamp it and count a timing fault.
  3. If clear_mask is nonzero, clear both LOW and HIGH pins for changed channels, then wait step.dead_time cycles.
  4. Apply the final target state with clr_mask followed by set_mask.
  5. Busy-wait until next_edge, or count an overrun if the edge has already passed.
Dead-time selection: step.dead_time uses g_dead_time_cycles_up when any changed channel turns HIGH in the target segment. Otherwise it uses g_dead_time_cycles_down. The current code stores one dead-time value per segment, so mixed transitions in the same segment share the rising value if at least one changed channel rises.

Sequence for Getting Data

Data acquisition is not performed inside the interrupt-disabled playback helper. The signal loop only requests it at safe maintenance points, after a pattern playback batch finishes.

  1. trigger_periodic_analog_read() first checks ble_an_read_state; disabled means return immediately.
  2. g_cycle_count advances modulo g_cycle_nrun.
  3. When the counter wraps to zero, the BLE analog state becomes READING.
  4. The loop gives sem_analog_read_trigger, allowing the analog read task to do the slower work outside the hot playback section.
  5. When fresh ADC values later arrive, g_adc_fresh lets update_control_corrections() consume them before the next playback batch.

Sequence for Selecting Data

The signal data is double-buffered. The running loop reads one active dataset while command handlers write the inactive dataset. The swap is requested with an atomic flag and applied only between playback batches.

Entry pointWhat it writesHow selection is made
signal_update_from_string() Parses a payload into time_durations[] and modes_d4/d5/d6[]. Writes g_dataset_b when A is active, otherwise writes g_dataset_a.
signal_set_alpha() Uses helper_set_dataset_from_alpha() to generate a dataset from alpha. Uses the same inactive-buffer rule, then marks the swap pending.
apply_pending_dataset_swap() Changes the loop context pointer and g_active_set. Runs from signal_loop_task before control correction and playback, never mid-pattern.

Loop Walkthrough

  1. signal_loop_task starts with dataset A, clears the correction buffers, and turns the LED on.
  2. Each iteration checks for a pending dataset swap and applies it before signal generation.
  3. If control is enabled and fresh ADC data exists, the loop computes duration corrections from ADC error and the active dataset gain matrix.
  4. The nominal pattern duration decides whether to batch short patterns with SHORT_PATTERN_BATCH_CYCLES repeats or run once.
  5. execute_signal_pattern disables interrupts, drives GPIO registers from precomputed masks, and records timing counters.
  6. After playback, the task may trigger analog acquisition, updates the timing snapshot, and repeats while the signal state remains RUNNING.
  7. When stopped, all six complementary outputs are forced low, the LED is turned off, and the task deletes itself.

Why This Shape Exists

The design separates slow work from precise work. Parsing, alpha generation, matrix math, and step precompute happen outside the interrupt-disabled GPIO block. The hot path only clears changed outputs, waits the configured dead-time cycles, asserts the precomputed target state, and waits for the next absolute edge. That keeps the waveform deterministic while still allowing BLE updates, analog sampling, and control correction between playback batches.
