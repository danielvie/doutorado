Problem Statement

The project addresses systems whose actuators are inherently switched. These systems are useful, but control is difficult because the actuator cannot change continuously and each command must respect timing constraints.

Formal Plant Model

The thesis writes the plant as a continuous-time switched affine system:

dx/dt = A_sigma(t) x(t) + b_sigma(t)
x(t) in R^n
sigma(t) in {1, 2, ..., M}
Problem elementMeaning in the project
Switched actuatorThe input is a mode or switch command, not a continuously variable signal.
Dwell-time constraintEach mode must remain active for at least a minimum duration.
Target behaviorA periodic trajectory or limit cycle, not only an equilibrium point.
Control variablePerturbations delta t applied to nominal switching instants.
Problem consequence: The controller must shape timing, not only amplitude. That is why the method works with switching-time perturbations.

Modeling Method

The modeling is conducted in two stages: first define a nominal periodic trajectory, then derive local cycle-to-cycle error dynamics around that trajectory.

1. Target Periodic Trajectory

Chapter 2 searches for a cycle that minimizes oscillation around an average reference while satisfying period and dwell-time constraints.

minimize integral_0^Tp (x(t) - xbar)^T Q (x(t) - xbar) dt
subject to:
  x(0) = x(Tp)
  Tp <= Tp_max
  t_i - t_{i-1} >= Delta t_i,min

2. Mode Sequence And Dynamics

The model allows the dynamics to change after each switching command. A mode sequence can reuse the same dynamics at different intervals, which is needed for converter examples.

ExampleState modelPurpose
Double integratorSame A matrix, different affine terms b_i.Validation case against prior work.
Buck-boost converterTwo operation modes with capacitor voltage and inductor current states.Changing dynamics at switching times.
Multilevel converterEight switch combinations with repeated modes in one cycle.High-complexity converter case.
Hardware article converterThree-cell flying capacitor converter with u1,u2,u3 switch commands.Physical platform for switched-system experiments.

3. Converter Model Used In Article And Thesis

The multilevel converter state is built from flying capacitor voltages and load current. The thesis uses affine dynamics; the article appendix also presents an augmented autonomous form.

x = [vc1, vc2, iL]^T
A_i = [[0, 0, (u2-u1)/C1],
       [0, 0, (u3-u2)/C2],
       [(u1-u2)/L, (u2-u3)/L, -R/L]]
b_i = [0, 0, (E/L) u3]^T
Modeling decision: The nominal cycle is not an afterthought; it is an input to the controller. The MPC regulates deviation from this precomputed cycle.

MPC Design

The MPC design converts deviations in switching instants into a constrained quadratic program. The manipulated variable is delta t, the perturbation from nominal switching times.

Prediction Equation

The linearization relates the error at the end of a cycle to the current error and the timing perturbation. In MPC notation, x becomes e and the control input becomes delta t.

e[j+1] = Phi e[j] + Gamma delta_t[j]
J = ||e[Np]||^2_Pf + sum_j ||e[j]||^2_Q + ||delta_t[j]||^2_R

Dwell-Time Constraints

The dwell-time inequalities are rearranged into matrix form so they can be passed to the constrained optimizer.

t_i - t_{i-1} >= Delta t_i,min
Delta tbar_i = tbar_i - tbar_{i-1}
L delta_t >= c
-S delta_t <= -c  # optimizer-compatible form
Design choiceReason
Linearize around reference cycleKeeps the predictive problem local and convex near the target trajectory.
Use switching-time perturbationsMatches the physical actuator: the controller changes when switches occur.
QP formulationKeeps the problem in a mature convex optimization class.
Feasibility regionsShows which initial errors can be handled for a chosen prediction horizon.
Control interpretation: The MPC does not choose arbitrary switch sequences; it adjusts the timing of a known reference switching cycle while respecting dwell time.

Results Obtained

The reported results show the method working across increasing complexity: a baseline double integrator, a buck-boost converter, and a multilevel converter.

Result areaReported outcomeInterpretation
Double integratorWithout MPC, the trajectory diverges from the target cycle; with MPC over 50 cycles, it is driven to the target cyclic trajectory.Validates the extended formulation against a simpler known case.
Buck-boost converterBoth open-loop and MPC trajectories converge, but MPC converges faster to the target cyclic trajectory.For stable dynamics, MPC improves transient convergence.
Multilevel converterOpen-loop simulations converge to different limit cycles depending on initial condition; MPC drives the state to the desired target cycle.The controller matters most in the more complex converter case.
Hardware article validationSwitch commands for alpha=0.4 and T=282 us were captured; output resistor voltage changes after alpha changes from 0.4 to 0.8.The physical platform generates measurable switched behavior suitable for control experiments.
