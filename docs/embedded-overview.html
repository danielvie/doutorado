Observations

The major risks are generated-output sprawl and environment-specific commands.

ObservationImpact
esp32_idf/ includes build/cache/managed output.Navigation and version control can become noisy.
bun.lock and package-lock.json both exist.Package manager choice is ambiguous.
Taskfiles include user-machine paths and ports.Commands may fail on another machine without edits.
Generated proto files exist in firmware and web.Schema ownership and regeneration order should be documented.
PCB outputs and backups are stored with sources.Manufacturing outputs need source-of-truth rules.
Practical next step: Make generated files, board releases, and local-only caches explicit in docs or ignore rules before broad cleanup.
