4. Expected Behavior and Limits

The expected result is a stable standard output edge that can be aligned in 0.1 us steps without changing the stored signal duration model.

Area Expected result
Fixed standard-edge offset Tunable with g_signal_edge_overhead_cycles through signal.set_edge_overhead.
Flash/cache variation Reduced because key timing functions are marked IRAM_ATTR.
Dead-time output Remains separate from the standard-edge compensation path.
Stored durations Remain represented as 0.1 us ticks.
Debug visibility debug.signal_timing includes "eo" for the active edge overhead.

This is still a software-timed GPIO loop. If future tests require hardware-enforced timing under broader BLE, ADC, or system-load conditions, the next design step should be an ESP32 waveform peripheral such as RMT or MCPWM.

Acceptance criterion: after calibration, the standard output edge should align to the requested timing within the 0.1 us target on the oscilloscope, and repeated loops should remain stable.
