1. Overview

This document explains the firmware change that makes standard GPIO signal edges more robust while preserving the project's 0.1 us timing model.

Fact Meaning
CPU = 240 MHz One CPU cycle is approximately 4.1667 ns.
24 cycles Equivalent to 0.1 us.
48 cycles Equivalent to 0.2 us.

The change treats the oscilloscope result as a stable fixed edge-placement offset, not as random jitter. The firmware now exposes that offset as a cycle-level compensation value.

2. Implementation

The implementation keeps the existing software-timed signal loop and adds the smallest calibration surface needed for the measured problem.

File Change
main/include/helper_common.h helper_delay_cycles() is marked IRAM_ATTR.
main/include/signal_controller.h Declares the global edge overhead compensation knob.
main/src/signal_controller.cpp Applies compensation in the hot signal loop, marks key timing functions as IRAM_ATTR, snapshots control state once, and reports "eo" in timing debug JSON.
main/src/ble_ui_command_router.cpp Adds the signal.set_edge_overhead command.

Global compensation value

Show code: cpp
volatile uint32_t g_signal_edge_overhead_cycles = 0;

Standard edge scheduling

The requested edge remains anchored to the configured segment duration. The transition start is moved earlier by the configured overhead.

Show code: cpp
const uint32_t transition_deadline =
              next_edge - dead_time - edge_overhead;

Runtime command

Show code: json
signal.set_edge_overhead {"cycles":24}

Dead-time compensation and standard-edge overhead are intentionally separate. The new value should correct visible standard edge placement only.

3. Calibration Workflow

Calibration is a short oscilloscope-driven loop: measure the stable offset, convert it to cycles, set the overhead, and re-measure.

Oscilloscope observation Suggested command Reason
Standard edge is stable at about -0.1 us signal.set_edge_overhead {"cycles":24} 24 cycles at 240 MHz equals 0.1 us.
Standard edge is stable at about -0.2 us signal.set_edge_overhead {"cycles":48} 48 cycles at 240 MHz equals 0.2 us.
Edge overshoots after compensation Reduce cycles in small steps. The configured overhead is larger than the measured fixed offset.
  1. Generate the same standard signal pattern that showed the offset.
  2. Measure the visible GPIO edge on the oscilloscope.
  3. Set signal.set_edge_overhead to the nearest cycle value.
  4. Run debug.signal_timing and confirm the "eo" field.
  5. Re-measure. The oscilloscope is the acceptance test.

4. Expected Behavior and Limits

The expected result is a stable standard output edge that can be aligned in 0.1 us steps without changing the stored signal duration model.

Area Expected result
Fixed standard-edge offset Tunable with g_signal_edge_overhead_cycles through signal.set_edge_overhead.
Flash/cache variation Reduced because key timing functions are marked IRAM_ATTR.
Dead-time output Remains separate from the standard-edge compensation path.
Stored durations Remain represented as 0.1 us ticks.
Debug visibility debug.signal_timing includes "eo" for the active edge overhead.

This is still a software-timed GPIO loop. If future tests require hardware-enforced timing under broader BLE, ADC, or system-load conditions, the next design step should be an ESP32 waveform peripheral such as RMT or MCPWM.

Acceptance criterion: after calibration, the standard output edge should align to the requested timing within the 0.1 us target on the oscilloscope, and repeated loops should remain stable.
